### 3. Chrome 自动化模式配置
**时间:** 08:50

用户询问为何无法使用真实 Chrome 浏览器。

**根本原因:**
Chrome 需要带远程调试端口启动才能被自动化控制。

**解决方案:**
创建桌面快捷方式 `Chrome-自动化模式.command`:
```bash
#!/bin/bash
# 关闭现有 Chrome
osascript -e 'tell app "Google Chrome" to quit'
sleep 2
# 启动带调试端口的 Chrome
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222
echo "Chrome 已启动在自动化模式 (端口 9222)"
```

**使用方法:**
1. 关闭现有 Chrome（如果有）
2. 双击运行桌面上的 `Chrome-自动化模式.command`
3. 正常浏览网页
4. 需要自动化控制时会自动连接到此 Chrome 实例

## 技术要点

### OpenClaw 模型配置注意事项
1. **API 类型必须准确**: 本地模型使用 `openai-completions` 而非 `openai`
2. **配置验证**: 使用 `openclaw config get agents.defaults` 检查配置
3. **重启应用**: 修改配置后需要 `openclaw gateway restart` 或 `SIGUSR1` 信号
4. **别名设置**: 在 `agents.defaults.models` 中设置，方便 `/model` 命令使用
