# 消毒与病媒生物管理平台 - 系统架构设计

## 1. 平台概述

### 1.1 平台名称
**柳州市疾病预防控制中心消毒与病媒生物管理平台**

### 1.2 建设目标
构建一个集**传染病消毒管理**、**病媒生物监测**、**数据分析**、**报告生成**于一体的综合管理平台，实现病媒监测数据的信息化、标准化和智能化管理。

## 2. 模块详细设计

### 2.1 消毒管理模块（已存在 - 病原微生物防控系统）

**状态：** ✅ 已有系统（v1.5.0）

**功能：**
- 消毒剂配比计算
- 传染病消毒方案查询（40种法定传染病）
- 消毒记录管理
- 常用方案保存
- 离线使用支持（PWA + APK）

**技术栈：** HTML5 + JavaScript + Capacitor

**整合方式：** 作为平台子系统嵌入，统一用户认证

---

### 2.2 病媒监测模块（核心新建模块）

#### 2.2.1 监测类型总览

| 监测类型 | 监测方法 | 数据格式 | 频率 |
|---------|---------|---------|------|
| **蚊密度监测** | 诱蚊灯法 | 布灯数、诱蚊夜数、捕获数 | 每月 |
| **蝇密度监测** | 捕蝇笼法 | 布笼数、诱蝇天数、捕获数 | 每月 |
| **蟑密度监测** | 粘蟑法 | 布放数、回收数、粘捕数 | 每月 |
| **鼠密度监测** | 粘捕法 | 布放数、回收数、粘捕数 | 每月 |
| **蜱虫监测** | 宿主寄生蜱/游离蜱 | 监测点、宿主种类、蜱数量 | 按方案 |
| **伊蚊监测-布雷图** | 布雷图指数法 | 户数、积水数、阳性容器 | 每月 |
| **伊蚊监测-叠帐** | 双层叠帐法 | 帐数、诱蚊时长、捕获数 | 每月 |
| **伊蚊监测-诱卵器** | 诱蚊诱卵器法 | 放置数、回收数、阳性数 | 每月 |

#### 2.2.2 数据录入子模块

**功能设计：**
1. **Excel批量导入**
   - 支持标准模板导入
   - 数据校验（完整性、逻辑性）
   - 错误数据提示与修正

2. **在线表单录入**
   - 分步骤向导式录入
   - 实时计算（密度、指数等）
   - GPS定位自动获取监测点
   - 照片上传（监测现场、标本）

3. **移动端录入**
   - 离线录入、联网同步
   - 扫码快速定位监测点
   - 语音输入支持

#### 2.2.3 监测点管理

- 监测点GIS地图展示
- 监测点基础信息管理
- 监测历史记录查询
- 监测点分类标签（城区、农村、特殊环境等）

### 2.3 数据分析模块

#### 2.3.1 实时数据看板

**核心指标：**
- 当月各类病媒密度/指数
- 同比、环比变化趋势
- 风险等级预警（根据国家标准）
- 监测完成率统计

**可视化图表：**
- 密度季节消长曲线图
- 城区分布对比图
- 环境类型构成图
- 达标情况统计图

#### 2.3.2 趋势分析

- 年度趋势分析
- 季节性规律分析
- 长期趋势预测（时间序列分析）
- 异常数据预警

#### 2.3.3 对比分析

- 城区间对比
- 环境类型对比
- 年度间对比
- 与国家标准对比

### 2.4 报告中心模块

#### 2.4.1 报告类型

| 报告类型 | 生成周期 | 主要内容 |
|---------|---------|---------|
| **月报** | 每月 | 当月监测数据汇总、密度统计、风险分析 |
| **季报** | 每季度 | 季度趋势分析、重点地区分析 |
| **年报** | 每年 | 全年监测总结、季节消长规律、防治建议 |
| **专项报告** | 按需 | 登革热媒介监测、蜱虫监测等专项分析 |
| **工作总结** | 按需 | 年度/专项工作总结报告 |

#### 2.4.2 报告生成功能

- **一键生成：** 选择模板、时间范围，自动生成报告
- **智能填充：** 自动提取数据、生成图表、插入分析结论
- **人工编辑：** 支持在线编辑、批注、修订
- **多格式导出：** Word、PDF、PPT

#### 2.4.3 报告模板管理

- 模板自定义配置
- 模板版本管理
- 模板权限控制

## 3. 数据库设计

### 3.1 核心数据表

```sql
-- 监测点表
CREATE TABLE monitoring_points (
    id INT PRIMARY KEY,
    point_code VARCHAR(50),      -- 监测点编码
    point_name VARCHAR(200),     -- 监测点名称
    district_id INT,             -- 所属区县
    address VARCHAR(500),        -- 详细地址
    env_type VARCHAR(50),        -- 环境类型
    latitude DECIMAL(10,7),      -- 纬度
    longitude DECIMAL(10,7),     -- 经度
    point_type VARCHAR(50),      -- 监测点类型
    status INT,                  -- 状态
    create_time DATETIME
);

-- 蚊密度监测表（诱蚊灯法）
CREATE TABLE mosquito_density (
    id INT PRIMARY KEY,
    monitor_date DATE,           -- 监测日期
    point_id INT,                -- 监测点ID
    month INT,                   -- 月份
    lamp_count INT,              -- 布灯数
    night_count INT,             -- 诱蚊夜数
    female_count INT,            -- 捕获雌蚊数
    density DECIMAL(10,2),       -- 密度（计算值）
    species VARCHAR(200),        -- 蚊种组成
    weather VARCHAR(100),        -- 天气情况
    remark TEXT,
    create_time DATETIME
);

-- 布雷图指数监测表
CREATE TABLE breteau_index (
    id INT PRIMARY KEY,
    monitor_date DATE,           -- 监测日期
    point_id INT,                -- 监测点ID
    household_count INT,         -- 调查户数
    water_count INT,            -- 积水容器数
    positive_count INT,         -- 阳性容器数
    breteau_index DECIMAL(10,2), -- 布雷图指数（计算值）
    -- 各类积水容器明细...
    create_time DATETIME
);

-- 四害监测汇总表
CREATE TABLE vector_monitoring (
    id INT PRIMARY KEY,
    vector_type VARCHAR(50),     -- 病媒类型（蚊/蝇/蟑/鼠/蜱）
    monitor_method VARCHAR(50),  -- 监测方法
    monitor_date DATE,           -- 监测日期
    point_id INT,                -- 监测点ID
    district_id INT,             -- 区县ID
    month INT,                   -- 月份
    year INT,                    -- 年份
    raw_data JSON,               -- 原始数据（JSON格式）
    density DECIMAL(10,4),       -- 密度值
    risk_level VARCHAR(20),      -- 风险等级
    create_time DATETIME
);

-- 报告模板表
CREATE TABLE report_templates (
    id INT PRIMARY KEY,
    template_name VARCHAR(200),  -- 模板名称
    template_type VARCHAR(50),   -- 模板类型
    content TEXT,                -- 模板内容
    parameters JSON,             -- 参数配置
    status INT,
    create_time DATETIME
);

-- 生成的报告表
CREATE TABLE generated_reports (
    id INT PRIMARY KEY,
    template_id INT,             -- 模板ID
    report_title VARCHAR(500),   -- 报告标题
    report_type VARCHAR(50),     -- 报告类型
    start_date DATE,             -- 开始日期
    end_date DATE,               -- 结束日期
    content TEXT,                -- 报告内容
    file_path VARCHAR(500),      -- 文件路径
    status INT,
    create_time DATETIME
);
```

## 5. 开发计划

### 5.1 阶段划分

| 阶段 | 时间 | 主要任务 | 交付物 |
|------|------|---------|--------|
| **第一阶段** | 1-2月 | 基础平台建设、数据库设计、四害监测模块 | 可运行的基础版本 |
| **第二阶段** | 2-3月 | 伊蚊监测模块（三种方法）、数据分析模块 | 完整监测功能 |
| **第三阶段** | 3-4月 | 蜱虫监测模块、报告中心模块 | 报告生成功能 |
| **第四阶段** | 4-5月 | 移动端开发、系统集成测试 | 完整平台 |
| **第五阶段** | 5-6月 | 数据迁移、培训上线、优化迭代 | 正式上线 |

### 5.2 优先级排序

**高优先级（核心功能）：**
1. 基础数据库建设
2. 四害监测数据录入与查询
3. 基础数据分析与图表
4. 月报/年报自动生成

**中优先级（扩展功能）：**
5. 蜱虫监测模块
6. 伊蚊专项监测模块
7. 移动端应用
8. 数据大屏

**低优先级（增值功能）：**
9. 智能预警
10. 高级数据分析（AI预测）
11. 与其他系统对接

## 6. 风险与对策

| 风险 | 对策 |
|------|------|
| 历史数据迁移复杂 | 开发专门的数据清洗和导入工具 |
